# RustyGate v0.1.0

Release date: 2026-05-03

## Baseline capabilities

- `GET /health` and `GET /ready`
- `POST /v1/chat/completions` with validation, provider routing, and optional SSE streaming
- Real provider support for `openai_compatible` and `anthropic`
- Same-provider retries and fallback across retryable provider errors
- Provider circuit breaker state with half-open recovery probes
- Gateway bearer authentication for chat, model discovery, stats, and metrics endpoints
- In-memory global and per-key rate limiting with `Retry-After`
- Request body size, message count, and message content guardrails
- `GET /stats` and `GET /stats/providers` aggregate metrics
- Prometheus-compatible `GET /metrics`
- `GET /v1/models` model discovery with configured aliases
- Optional priority, cost-aware, and latency-aware routing policies
- Structured JSON errors with request IDs
- Optional SQLite persistence for request logs and provider attempts

All checks passed.

## Known limitations in v0.1.0

- Gateway auth uses one configured bearer token, not multi-key management or rotation.
- Rate limiting is in-memory per process, not distributed across replicas.
- `/ready` is process readiness only; it does not perform provider health checks.
- SQLite persistence has no retention or cleanup controls.
- No OpenTelemetry traces or external dashboard manifests.
- No Kubernetes manifests, Redis, semantic caching, web dashboard, multi-user billing, or full OpenAI API compatibility.

## Smoke script usage

```sh
export RUSTYGATE_GATEWAY_API_KEY=local-dev-gateway-key
./scripts/smoke.sh
```

Use a custom endpoint with:

```sh
BASE_URL=http://127.0.0.1:8080 ./scripts/smoke.sh
```
